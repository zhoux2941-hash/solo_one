# 影视剧组通告管理系统

## 项目概述
一个基于 Vue3 + Vite + Spring Boot + MySQL + Redis 的影视剧组通告管理系统。

## 技术栈
- **前端**：Vue 3 + Vite + Element Plus + Vue Router + Axios
- **后端**：Spring Boot 3.2 + Spring Security + JPA + JWT
- **数据库**：MySQL 8.0
- **缓存**：Redis

## 角色功能
1. **导演**：创建发布通告，选择演员、场景、时间
2. **演员**：查看个人日程表，系统自动检测时间冲突
3. **场务**：查看当日通告，确认物资是否备齐

## 环境要求
- JDK 17+
- Maven 3.6+
- Node.js 18+
- MySQL 8.0+
- Redis 6.0+

## 快速开始

### 1. 启动 MySQL
```sql
-- 创建数据库
CREATE DATABASE crew_notice DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

修改 `backend/src/main/resources/application.yml` 中的数据库配置：
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/crew_notice
    username: your_username
    password: your_password
```

### 2. 启动 Redis
确保 Redis 在本地 6379 端口运行（无需密码）。

### 3. 启动后端
```bash
cd backend
mvn clean install
mvn spring-boot:run
```
后端服务将在 `http://localhost:8080` 启动。

系统启动时会自动创建测试账号：
| 角色 | 用户名 | 密码 |
|------|--------|------|
| 导演 | director | 123456 |
| 演员 | actor1 | 123456 |
| 演员 | actor2 | 123456 |
| 演员 | actor3 | 123456 |
| 场务 | pa | 123456 |

### 4. 启动前端
```bash
cd frontend
npm install
npm run dev
```
前端服务将在 `http://localhost:5173` 启动。

## 核心功能说明

### 时间冲突检测
- 导演发布通告时，后端自动检测选中演员在该日期的时间段是否与已有通告重叠
- 冲突检测优先从 Redis 缓存读取演员当日通告，加速检测
- 若有冲突，返回具体演员及重叠的通告信息

### Redis 缓存策略
- 缓存键格式：`actor:notices:{actorId}:{date}`
- 缓存内容：演员当日的所有通告列表
- 过期时间：24 小时
- 发布新通告或确认物资时自动失效相关缓存

### 时间冲突判断逻辑
两个时间段 [A开始, A结束] 和 [B开始, B结束] 存在冲突当且仅当：
- A结束 > B开始 **且** A开始 < B结束

即：新通告不是完全在已有通告之前，也不是完全在已有通告之后。

## API 接口

### 认证
- `POST /api/auth/login` - 用户登录
- `GET /api/auth/me` - 获取当前用户信息

### 用户
- `GET /api/users/actors` - 获取所有演员列表

### 通告
- `POST /api/notices` - 创建通告（导演）
- `GET /api/notices/by-date?date=2024-01-01` - 按日期查询通告
- `GET /api/notices/my` - 查询当前用户的所有通告（演员）
- `GET /api/notices/my?date=2024-01-01` - 查询当前用户某日的通告（演员）
- `GET /api/notices/{id}` - 查询通告详情
- `PUT /api/notices/{id}/confirm-materials` - 确认物资（场务）
